# E2E UAT — agent-executed walkthrough (CLI · REST/proxy · `--render` TUI)

> **Executed by an AI agent.** Every step is a copy-paste command plus a
> machine-checkable assertion (exit code / `jq -e` / `grep -q` / HTTP status).
> No step says "eyeball it" — if a check can't be automated it is rewritten
> until it can, or marked ⏭️ with the reason. The plan is host-independent:
> **Setup** builds an isolated sandbox; **Fixture selection** discovers test
> models from disk. Reset the **Findings log** + **Run log** (or copy this file
> to a dated run file) before each pass.

## What is under test

Two **surfaces**, both verified programmatically:

- **Human-facing rendering** — colored/padded TTY tables and the `--render`
  TUI frames. The agent exercises TTY-gated output via the `pty` helper and
  asserts on the captured text (it does not need a real terminal).
- **Machine surface** — `--json` shapes, exit codes, and the REST/proxy
  endpoints. Asserted with `jq -e`, `$?`, and `curl` status codes.

**NOT used:** `cargo test`, the integration/e2e suite, the `--features uat`
subcommand. This reproduces what a real client hits against a release-shaped
binary; green CI can hide a broken binary, which is the gap this plan covers.

## Execution protocol (for the agent)

1. Run **§Setup**, then **§Fixture selection**. Abort the pass if the binary
   won't build or no usable fixture model is found (record in Run log).
2. Execute **§0 → §15** in order. For each item: run the command(s), evaluate
   the assertion, mark the checkbox per the legend, and on ❌/⚠️ append a row
   to the **Findings log** and reference its ID inline.
3. Prefer non-destructive ordering; launches in §5–8 reuse the sandbox daemon.
4. Run **§Teardown** at the end (and on early abort).
5. An assertion that throws (`set -e` / `jq -e` non-zero / `[ ]` false) ⇒ ❌.
   A pass that needed a documented deviation ⇒ ⚠️ with the deviation noted.

## Status legend

`- [ ]` pending · `- [x] ✅` pass · `- [x] ❌` fail (see Findings) ·
`- [x] ⏭️` skipped/blocked (reason inline) · `- [x] ⚠️` pass-with-caveat.

## Setup (reproducible bootstrap)

```bash
set -uo pipefail
cd "$(git rev-parse --show-toplevel)"
cargo build --bin llamastash            # release shape: add --release, adjust BIN
export BIN="$PWD/target/debug/llamastash"
export GIT_SHA="$(git rev-parse --short HEAD)"
"$BIN" --version; echo "git: $GIT_SHA"

# Isolated sandbox — never touches the user's real daemon/config.
export UAT_ROOT="${TMPDIR:-/tmp}/uat-llamastash"
rm -rf "$UAT_ROOT"; mkdir -p "$UAT_ROOT"/{state,config,cache}
export LLAMASTASH_STATE_DIR="$UAT_ROOT/state"
export LLAMASTASH_CONFIG_DIR="$UAT_ROOT/config"
export LLAMASTASH_CACHE_DIR="$UAT_ROOT/cache"
: "${HF_HOME:=$HOME/.cache/huggingface}"; export HF_HOME   # real cache = models to find

command -v llama-server >/dev/null || export LLAMASTASH_LLAMA_SERVER="$HOME/.local/bin/llama-server"

# --- Helpers (functions, so they work non-interactively) -------------------
pty()   { script -qec "$*" /dev/null; }                 # run under a pty → exercise TTY-gated rendering
rt()    { cat "$LLAMASTASH_STATE_DIR/llamastash/runtime.json"; }   # control-plane url+token
ctl()   { local u t; u=$(rt|jq -r .url); t=$(rt|jq -r .token)
          curl -s -H "Authorization: Bearer $t" -H 'content-type: application/json' -d "$1" "$u/rpc"; }
PROXY() { "$BIN" status --json | jq -r '.proxy.listen' | sed 's#^#http://#'; }   # BASE=$(PROXY)
# HTTP helpers: print status code; body saved to $HTTP_BODY
http()  { local m=$1 url=$2; shift 2; HTTP_BODY=$(mktemp)
          curl -s -o "$HTTP_BODY" -w '%{http_code}' -X "$m" "$url" "$@"; }
jchat() { jq -nc --arg m "$1" --arg p "${2:-hi}" \
          '{model:$m,messages:[{role:"user",content:$p}],max_tokens:16}'; }
```

> **First-run confirmations** (do once, record results in §1.2 / §3.3):
> the exact `runtime.json` path and field names (`url`/`token`), and the real
> `list --json` row schema. Adjust `rt()`/`ctl()` and fixture filters if they
> differ from the assumptions above.

## Fixture selection (discovery-driven)

```bash
ROOTS=("$HF_HOME/hub" /mnt/work/lmstudio-models "$HOME/.lmstudio/models")
pick() { find -L "${ROOTS[@]}" -name '*.gguf' 2>/dev/null -printf '%s\t%p\n' \
         | grep -Eiv 'mmproj' \
         | { [ -n "$1" ] && grep -Ei "$1" || cat; } \
         | { [ -n "$2" ] && grep -Eiv "$2" || cat; } \
         | sort -n | head -1 | cut -f2; }
ref() { basename "$1" .gguf; }
export CHAT_REF="$(ref "$(pick '' 'embed|rerank')")"
export EMBED_REF="$(ref "$(pick 'embed' 'rerank')")"
export RERANK_REF="$(ref "$(pick 'rerank' '')")"
export SPLIT_REF="$(ref "$(find -L "${ROOTS[@]}" -name '*-00001-of-*.gguf' 2>/dev/null | head -1)")"
export AMBIG_REF="$("$BIN" list 2>/dev/null | grep -oiE '[a-z0-9.]+-[0-9]+b' | sort | uniq -d | head -1)"
printf 'CHAT=%s\nEMBED=%s\nRERANK=%s\nSPLIT=%s\nAMBIG=%s\n' \
  "$CHAT_REF" "$EMBED_REF" "$RERANK_REF" "$SPLIT_REF" "$AMBIG_REF"
# Gate: chat fixture must resolve to exactly one model.
"$BIN" show "$CHAT_REF" >/dev/null 2>&1 && echo "CHAT_REF ok" || echo "ABORT: tighten CHAT_REF"
```

> If a role has no local model → mark that section ⏭️ or `pull` a small one
> (§13.3). **Do-not-launch:** some local models never reach `/health` on a
> given backend (e.g. large MoE on an APU); use those only for `show`/`list`
> identity tests, never as a launch fixture. `$SPLIT_REF` is inspect-only.
