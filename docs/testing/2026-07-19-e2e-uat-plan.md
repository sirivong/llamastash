# E2E UAT ‚Äî agent-executed walkthrough (CLI ¬∑ REST/proxy ¬∑ `--render` TUI)

> **Target version: `0.1.0`** (built from `main`, git `d043f99`). Extends the
> v0.0.2 plan `docs/testing/2026-05-30-e2e-uat-plan.md` (which covers ¬ß0‚Äì¬ß15 there).
> This plan adds the surface shipped 0.0.2 ‚Üí 0.1.0 in **¬ß16‚Äì¬ß25**. Every step is a
> copy-paste command plus a machine-checkable assertion (exit code / `jq -e` /
> `grep -q` / HTTP status). No step says "eyeball it" ‚Äî if a check can't be automated
> it is rewritten until it can, or marked ‚è≠Ô∏è with the reason. The plan is
> host-independent: **Setup** builds an isolated sandbox; **Fixture selection**
> discovers test models from disk. Reset the **Findings log** + **Run log** (or copy
> this file to a dated run file) before each pass.

## What is under test

Three **surfaces**, all verified programmatically:

- **Human-facing rendering** ‚Äî colored/padded TTY tables and the `--render` TUI frames. The agent exercises TTY-gated output via the `pty` helper and asserts on the captured text (it does not need a real terminal).
- **Machine surface** ‚Äî `--json` shapes, exit codes, and the REST/proxy endpoints. Asserted with `jq -e`, `$?`, and `curl` status codes.
- **Control-plane IPC** ‚Äî `POST /rpc` (bearer) for `list_models` / `status` / `start_model` / `stop_model` / `presets_*` / `pull`, used to cross-check the CLI surface and to assert the IPC‚ÜîCLI contract.

**NOT used:** `cargo test`, the integration/e2e suite, the `--features uat` subcommand. This reproduces what a real client hits against a release-shaped binary; green CI can hide a broken binary, which is the gap this plan covers.

## Execution protocol (for the agent)

1. Run **¬ßSetup**, then **¬ßFixture selection**. Abort the pass if the binary won't build or no usable fixture model is found (record in Run log).
2. Execute **¬ß0 ‚Üí ¬ß25** in order. For each item: run the command(s), evaluate the assertion, mark the checkbox per the legend, and on ‚ùå/‚öÝÔ∏è append a row to the **Findings log** and reference its ID inline.
3. Prefer non-destructive ordering; launches in ¬ß5‚Äì¬ß9 reuse the sandbox daemon.
4. Run **¬ßTeardown** at the end (and on early abort).
5. An assertion that throws (`set -e` / `jq -e` non-zero / `[ ]` false) ‚áí ‚ùå. A pass that needed a documented deviation ‚áí ‚öÝÔ∏è with the deviation noted.

"$BIN" daemon start          # isolated daemon; IPC url+token land in runtime.json

Helpers (functions, non-interactive):
```bash
rt()  { "$BIN" "$@"; }                                   # run binary
rtt() { "$BIN" "$@" 2>&1 | tail -3; }                    # run + tail
ctl() { local t u; t=$(jq -r .ipc_token "$LLAMASTASH_STATE_DIR/runtime.json"); \
        u=$(jq -r .ipc_url "$LLAMASTASH_STATE_DIR/runtime.json"); \
        curl -s -H "Authorization: Bearer $t" -H "content-type: application/json" \
             -d "$2" "$u/rpc"; }                          # IPC json-rpc call
PROXY() { echo "http://127.0.0.1:11435"; }               # shared proxy port
```

### Scenario inputs (¬ß23‚Äì¬ß25)
These sections write **throwaway `config.yaml` / `state.json` files into the sandbox
`$LLAMASTASH_CONFIG_DIR` / `$LLAMASTASH_STATE_DIR`** and restart the isolated daemon ‚Äî
they never touch the user's real config. Restart pattern for a config swap:
```bash
cfg="$LLAMASTASH_CONFIG_DIR/config.yaml"
reload() { "$BIN" daemon stop >/dev/null 2>&1 || true; "$BIN" daemon start >/dev/null; "$BIN" status --json >/dev/null; }
```
Inputs used below:
- **`LLAMASTASH_DEBUG_FAKE_GPUS=2`** (debug build only) ‚Äî fans out **both** the host-metrics
  and the **launch** device catalog to 2 GPUs, so the multi-device / multi-GPU picker paths
  are exercisable on this single-GPU host (`src/backend/server.rs::debug_fake_multi_gpu`).
- **old-shape `config.yaml`** ‚Äî pre-0.1.0 flat keys (`jinja:`, `llama_server_path:`, a
  top-level `lemonade:` block) to prove the breaking migration (¬ß23).
- **removed-subkey `config.yaml`** ‚Äî a `backend.llamacpp.binary:` / `additional_binaries:`
  key (deleted; `deny_unknown_fields` should reject) (¬ß23).
- **missing-binary `config.yaml`** ‚Äî `backend.llamacpp.servers: [{binary: /nonexistent/llama-server}]`
  to fire the `server_binary_missing` doctor advisory (¬ß19).
- **auth `config.yaml`** ‚Äî `proxy.api_key: sk-uat-<rand>` to force data-plane auth on the
  loopback proxy (¬ß24). Auth is enforced whenever a key is set, independent of loopback.
