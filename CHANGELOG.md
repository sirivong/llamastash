# Changelog

All notable changes to LlamaStash will be documented in this file. The format follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/); the project intends to follow [SemVer](https://semver.org/spec/v2.0.0.html) starting with the first stable release.

## [Unreleased]

- Docs: ship an installable AgentSkills bundle under `skills/llamastash/` plus a Claude Code marketplace manifest under `.claude-plugin/marketplace.json` for harnesses that need to drive the LlamaStash CLI.
- Daemon: intelligent context auto-fit. When `ctx` is unset across every resolver layer (caller, last-params, YAML, built-in arch table), `start_model` now computes the largest context length that fits the current free VRAM ‚Äî or RAM, on CPU-only runs ‚Äî before spawning `llama-server`. The math reads the GGUF attention geometry, the file's tensor table for weight bytes, and the host-metrics snapshot, then solves `(free - weights - 1.5 GiB overhead) / (n_parallel * kv_per_token)`, clamped to `[4096, n_ctx_train]` and aligned to 256. The chosen value is emitted as `-c <N>` on the child argv (so llama.cpp's `--fit` becomes a no-op) and logged as `auto-fit ctx=<N> for <path>`. This replaces llama.cpp `--fit`'s 4096-floor collapse on Linux 7+ AMD iGPUs (Strix Halo) where the unified-memory free-space reading is broken. Empirical on a 64 GiB Strix Halo iGPU: Qwen3.6-27B-Q4_K_M now lands at `n_ctx=46848` (was 4096); Qwen3-reranker-0.6B-Q8_0 picks up its full 40960 native window. If the snapshot isn't ready or the GGUF lacks attention metadata, `ctx` is left unset and `--fit` still runs. An explicit `ctx` from the user, last-params, or YAML always wins.
- CLI: `daemon start` now detaches into the background by default and returns once the socket is bound. Bare `llamastash daemon start` previously held the terminal (foreground) which read as a hang from a fresh shell. The new path prints `daemon: starting in background‚Ä¶` ‚Üí `‚úì daemon: started (detached)` so the user sees progress, and `--foreground` (alias `-f`) keeps the daemon attached for systemd / runit / container `CMD` wrappers that own stdout/stderr. The re-exec'd child receives `--foreground` on its argv so it doesn't recursively detach (`start_detached` would otherwise fork-bomb after the default flip).
- Daemon: idle CPU regression fix. The host-metrics sampler used to run the full GPU vendor chain (`nvidia-smi` ‚Üí `rocm-smi` ‚Üí `system_profiler` ‚Üí `vulkaninfo`) on every 1 Hz tick ‚Äî ~86,400 subprocess spawns/day on an idle supervisor with no inference children, observed in the wild as ~60% steady-state CPU. The per-tick path now calls only the active vendor via `gpu::refresh_active`; the full chain runs once at sampler start and every 60 ticks (~1 min) to catch hotplug / late driver loads. CPU-only, Vulkan, and Metal hosts skip per-tick subprocess spawns entirely. Sampler also hoists `sysinfo::Components` out of the loop (was allocating a fresh list and double-refreshing every tick) and threads the shared `MetadataCache` into `ollama::enumerate` so the 5-minute periodic rescan stops re-parsing every Ollama blob.
- TUI: opt-in mouse focus + scroll. Enable via `mouse_focus: true` in `config.yaml` or the global `--mouse-focus` flag (OR-ed; aliasing `llamastash` to `llamastash --mouse-focus` is a clean always-on path). Left-click on the Models list, the right pane, or a tab label (`Settings`/`Logs`/`Chat`/`Embed`/`Rerank`) moves focus / switches tab. Wheel up/down is a verbatim replay of `‚Üë`/`‚Üì` in the current focus ‚Äî same per-pane semantics as the arrow keys, with no new contract to remember. Drag / Up / motion events are filtered at the input thread so a bypass-modifier text selection doesn't scrub the list and the event channel can't saturate under a moving cursor. Modal overlays (HF dialog, confirm popup, help) ignore mouse events so a stray press can't confirm a destructive action. Off by default so the terminal retains native click-and-drag text selection.
- TUI: dashboard split now 65/35 (Models / right pane) with 1-cell horizontal padding inside both pane borders, so content breathes against the border line.
- TUI: model-list section headers collapse the parent path to a short label ‚Äî `owner/repo` for HuggingFace and LM Studio caches, `Ollama` for blob storage, and the trailing path segments for user-configured `model_paths`.
- TUI: right pane shows the focused model's full file path under the model name in the theme's muted tone, hard-wrapped across up to 3 lines so narrow panes still surface the location instead of a truncation stub. `$HOME` collapses to `~`.
- Makefile: `make run <args>` now forwards extra goals to `cargo run --`, so `make run list` / `make run start <model>` work without further plumbing. New `make render` target renders the TUI at a sweep of representative sizes for layout review.
- TUI: keybinding model split into `hint` (short UI-chip text) and `description` (longer help-overlay text, optional, falls back to `hint`). Each binding now declares its help-overlay home via a `categories: &[Category]` list, replacing the hand-rolled `Row::Single`/`Row::Multi` constants. The help overlay derives its layout directly from `DEFAULT_BINDINGS` ‚Äî adding a new chord no longer requires a parallel edit in `help_overlay.rs`. Net `-220` lines.
- TUI: `Shift+P` (pull-from-HF) now fires from the right pane too, not just the models list. Scoped to NAV so input fields keep typing capital `P` as text.
- TUI: help overlay grouping pruned ‚Äî `General` absorbs pane navigation (Tab / Shift-letter quick-jumps), motion, edit, Shift+Enter newline, and a consolidated `Esc` row ("cancel/clear/back"). Yank rows (`c/u/y/p`) collapse onto Models, Settings, and Logs (curl-only) instead of repeating under Chat/Embed/Rerank. `Ctrl+S` (stop) and `Ctrl+D` (delete) move out of General into their model surfaces. Chat / Embed / Rerank fold into a single `Chat/Embed/Rerank` section with `Enter:submit` and `Ctrl+R:toggle <think> blocks` ‚Äî per-tab variants and the rerank ‚Üë/‚Üì field-cycle hide from the overlay.
- TUI: help overlay scrolls ‚Äî `j`/`k` (and arrows / PgUp / PgDn / Home) slide the content vertically when a short terminal can't fit every section; a `j/k:scroll` chip joins the title bar when overflow exists. Narrow widths (< 80 cells of usable inner space) collapse to a single column so descriptions stop truncating.
- TUI: HF pull dialog's per-stage chords (`o` cycle sort, `n` next page, `p` previous page) now appear in the help overlay under `HF pull dialog` ‚Äî previously the dialog's own footer was the only discoverability surface.
- TUI: vim muscle memory at home ‚Äî list scroller picks up `Ctrl+F`/`Ctrl+B` page jumps, `Ctrl+U` half-page (collapses to page-up), `0`/`$` top/bottom aliases, and `i` opens the right-pane input alongside `e`. `gg` already worked incidentally (second `g` no-ops at the top). - TUI: adaptive layout. Dashboard floor drops from 80√ó20 to 60√ó20. Below 100 cells the right pane goes drill-in-only: hidden by default, opens when focus moves into it (`Enter` on a model row), and closes on `Esc` so the list expands back. Above 100 cells nothing changes ‚Äî both panes always render. - Makefile: new `make audit` maintainer target runs the full repo-audit suite in one pass: `make test`, release build, release-binary size capture, `cargo tree --duplicates`, `cargo audit --json`, `cargo geiger --all-targets --features test-fixtures`, and Tarpaulin XML output under `target/audit/`. `make audit-summary` prints the headline numbers from those artifacts, and Geiger output is captured even when the tool exits non-zero on upstream scan warnings.
- TUI: `Enter` on the Models list now branches by running state. Idle row: stage the launch picker and focus Settings (next `Enter` launches). Running row: focus the read-only running view *without* staging a picker ‚Äî the bottom strip leads with `e:edit for launch` so the user explicitly opts into a new launch. Pre-fix a stray `Enter` on a running row could silently dispatch a duplicate launch.
- OpenAI-compatible proxy router on `127.0.0.1:11435` by default (enabled out of the box) ‚Äî single URL, model-name routing across `/v1/models`, `/v1/chat/completions`, `/v1/completions`, `/v1/embeddings`, `/v1/rerank` with auto-start, single-flight launch coalescing, and family-MRU fallback on launch failure (substitution observable via `x-llamastash-served-by` + `x-llamastash-fallback-reason: launch_failed | family_mismatch` headers). Loopback only, no auth, no TLS. Surfaces through `status.proxy` (IPC + CLI `--json`) and the TUI Daemon info pane. Ships alongside the Ollama discovery surface ‚Äî `GET /api/tags`, `GET /api/version`, `GET /api/ps`, `POST /api/show` ‚Äî so Ollama-shape discovery libraries (`ollama-python` default path, `OLLAMA_HOST` env detection, IDE plugins that probe `/api/tags`) recognise llamastash and fall through to the OpenAI-compat endpoints for inference; Ollama's inference endpoints (`/api/chat`, `/api/generate`, `/api/embed`) are deferred (TODO ¬ßR2). `llamastash daemon start --proxy-port <PORT>` overrides the mode default for one process (use `--proxy-port 0` for an ephemeral port; the actual address is reported via `llamastash status --json | jq .proxy.listen`). See [`docs/usage.md ¬ßProxy (OpenAI-compatible listener)`](docs/usage.md#proxy-openai-compatible-listener) and [`docs/plans/2026-05-21-001-feat-proxy-router-plan.md`](docs/plans/2026-05-21-001-feat-proxy-router-plan.md).
- Proxy: Ollama drop-in mode (opt-in). New `proxy.ollama_compat: true` (config), `--ollama-compat` (CLI), or `LLAMASTASH_OLLAMA_COMPAT=1` (env) ‚Äî any one OR-ed in ‚Äî makes the proxy answer `GET /` with the byte-exact `"Ollama is running"` handshake the official `ollama` CLI and other Ollama-Go-based clients check for, and shifts the default port from `11435` ‚Üí `11434` (Ollama's well-known port). Default mode keeps the honest `"LlamaStash is running"` identity and the `11435` coexistence port so an installed Ollama daemon doesn't collide. Both modes scan up to `11440` for a free slot, and both speak the same `/v1/...` + Ollama-discovery surfaces. Pre-v0.0.1 schema break: `proxy.port` in `config.yaml` is now `Option<u16>` ‚Äî omitting it picks the mode default (was hard-coded `11434`).

## [0.0.1] ‚Äî Unreleased

First publicly-installable release. A single `llamastash` binary acts as TUI, CLI, and on-demand daemon for running local LLMs via [llama.cpp](https://github.com/ggml-org/llama.cpp). Distributed via Cargo, a Homebrew tap, and a GitHub-hosted install script, with a marketing site at [llamastash.cli.rs](https://llamastash.cli.rs).

### Zero-to-chat in one command

- `llamastash init` ‚Äî interactive first-run wizard that detects hardware (NVIDIA / AMD-ROCm / Apple Metal / Vulkan / CPU), installs the right `llama-server` variant, picks a starter GGUF tuned to your VRAM, downloads it, writes a tuned `config.yaml`, and smoke-launches. `--recommended` / `--only` / `--skip` / `--json` / `--offline` flags make it agent-friendly.
- `llamastash doctor` ‚Äî read-only health check with typed, agent-branchable findings and stable `fix_hint` pointers. Always exits `0`.
- Hardware-aware model recommender with a VRAM-fit filter plus composite ranking (benchmark √ó tok/s √ó params √ó recency), over a daily-CI-refreshed snapshot.

### Discovers what you already have

- Auto-scans HuggingFace, Ollama, and LM Studio caches plus user-configured paths; live filesystem watching surfaces new GGUFs without a restart.
- Rich GGUF intelligence ‚Äî header parser surfacing architecture, parameter count, quantization, native context, chat template, and reasoning hints. KV-cache-aware memory estimates that account for chosen context length.
- Smart deduplication ‚Äî symlinks collapse to their target, split GGUFs unify, Ollama content-addressed blobs surface under their human-readable name.

### Launches anything, supervises everything

- Daemon-on-demand over a `0600` Unix socket with peercred auth. First client auto-spawns; running models survive TUI close via three-factor orphan re-adoption (PID alive + port listening + `/v1/models` path match).
- Multi-model concurrency ‚Äî each launch gets its own port (auto-allocated from a configurable range) and a `Launching ‚Üí Loading ‚Üí Ready ‚Üí Stopping ‚Üí Stopped` state machine with `/health` probing.
- GPU-aware built-in arch-defaults table covering `llama*`, `qwen2*`, `qwen3*`, `mistral`, `mixtral`, `gemma*`, `phi*`, `deepseek*`, `granite`, `falcon`, `stablelm`, `command-r`, plus a `*` fallback. Fresh install gets sensible `n_gpu_layers` / `flash_attn` on every supported backend with zero YAML.
- Typed launch-knob editor with `(user)` / `(last used)` / `(arch default)` / `(model default)` / `(server default)` source chips. Layered resolver: `preset > last-params > yaml arch_defaults > built-in table > llama-server`.
- Named presets, favorites, and last-params recall persisted in `state.json`.

### A TUI that doesn't get in your way

- Keyboard-driven everywhere ‚Äî vim-style `hjkl`, `/` filter, `f` favorite, `u`/`c`/`p` yank URL/curl/path, `t` cycle theme, `?` contextual help.
- Right pane is your smoke test ‚Äî Logs / Chat / Embed / Rerank tabs hit the same OpenAI-compatible endpoints any external client would use.
- In-TUI HuggingFace browser (`d`) ‚Äî three-stage Search ‚Üí File picker ‚Üí Confirm modal over `/api/models`. Search, sort, paginate, per-file fit `‚úì` / `‚öÝ` / `‚úó`, sharded-set collapse, pinned download strip with `Ctrl+X` cancel and `Ctrl+D` delete-from-disk.
- Five built-in themes (Catppuccin Macchiato default + Latte, Gruvbox Dark, Solarized Dark, Monochrome) plus a `custom_theme` config block for user palettes.
- Every TUI action rebindable via a `keybindings:` config block with a kdash-style key-spec dialect. Destructive actions sit behind `Ctrl`; cross-pane navigation behind `Shift`. Unicode keycap glyphs in the help bar (`‚Üπ` / `‚á•` / `‚èé` / `‚áß` / `‚åÉ ‚å• ‚åò`).
- Accessible by default ‚Äî status indicators dual-encoded with colour + glyph; a "terminal too small" placeholder below 40√ó10.

### First-class CLI for agents and scripts

- Subcommands cover every TUI capability: `list`, `start`, `stop`, `status`, `logs`, `presets`, `favorites`, `last-params`, `daemon`, `init`, `doctor`, `pull`, `recommend`. Every read+mutation command supports `--json` as the agent contract.
- Documented exit codes per failure class (`66` ambiguous ref, `67` launch failure, `69` pull failure, `70` missing `llama-server`, `72`/`73`/`74` init phases). Pin numbers, not message text.
- Colored TTY output, byte-stable TSV when piped, `NO_COLOR` / `--no-colors` honored, `--json` byte-stable regardless.
- `llamastash pull <owner/repo[:filename]>` standalone HF fetch via `hf-hub` ‚Äî honours `HF_TOKEN`, refuses world-readable token cache files, performs disk-space precheck before any bytes hit disk.
- `llamastash recommend` exposes the wizard's recommender on its own. Reproducible pulls via `--revision <SHA>`.

### Built to be safe to run

- Unix-socket peercred auth (`0600`) ‚Äî only your own UID can drive the daemon. No tokens to manage; no network surface.
- Hardened fetch substrate ‚Äî HTTPS-only with host allowlist, redirect cap, body-size cap, IP-literal refusal. `--offline` / `LLAMASTASH_OFFLINE` short-circuits before any DNS.
- Archive-bomb defenses on installers ‚Äî entry-count / total-size / compression-ratio caps; refuses hardlink, symlink, absolute-path, or `..` entries. SHA-256 verified before extract from the GitHub Releases asset's `digest` field.
- Atomic, mode-checked config + state writes ‚Äî `0600` final mode, refuses symlinks and world-writable parents. Corrupt `state.json` quarantined to `state.json.broken-<ts>` rather than blocking daemon boot.
- Side-by-side daemons via `LLAMASTASH_STATE_DIR` / `LLAMASTASH_CONFIG_DIR` / `LLAMASTASH_CACHE_DIR` / `LLAMASTASH_SOCKET` overrides.

## How to read this file

LlamaStash is pre-1.0; the entire pre-release history is bundled under the first publishable tag, [0.0.1], rather than backfilled into a series of synthetic tags. The ledger starts there.
